Field Manual — /etc/hosts

Local name overrides for controlled testing, lab routing, and deterministic resolution.

Purpose

  • Force a hostname to resolve to a specific IP (lab/validation).
  • Stabilize tooling that depends on DNS names (virtual hosts, TLS SNI).
  • Temporary overrides when DNS changes aren’t possible or desirable.

Where it lives

  • Linux/macOS: /etc/hosts
  • Windows: C:\Windows\System32\drivers\etc\hosts

Operational notes

  • Changes are local-only to the current machine.
  • Prefer temporary changes; document additions in your log.
  • Flush DNS cache if resolution doesn’t update (varies by OS).

Example entry (safe)

# IP address    hostname(s)
192.0.2.10       app.local
192.0.2.11       api.app.local

Reinforcement

  • Track overrides to avoid “mystery routing” during future work.
  • Remove entries after engagement to prevent stale resolution.
