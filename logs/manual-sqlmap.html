Field Manual — sqlmap

SQL injection validation workflow notes for authorized testing and defensive reporting.

Scope & safety

  • Only test owned/authorized applications.
  • Define load constraints (avoid service disruption).
  • Use a staging environment when possible.

Validation methodology (high level)

  • Identify candidate inputs and confirm request context (GET/POST/JSON).
  • Confirm parameterization status and server-side query construction patterns.
  • Capture evidence (server errors, timing anomalies, logs) in a controlled manner.
  • Stop at validation: focus on proof + remediation guidance.

Evidence to capture

  • Request shape (sanitized) + response behavior
  • Server-side logs (query errors, WAF events)
  • App code location (handler/controller) if available

Reinforcement

  • Use parameterized queries / prepared statements
  • Validate and normalize input; enforce allowlists
  • Least privilege DB accounts; separate read/write roles
  • WAF/IDS signals + logging for detection

Public manual omits operational exploitation flags/recipes. Keep exact commands in private, client-authorized playbooks.
